# Prototype Instructions

Run the local server yourself and open the preview in the in-app browser. Do not give the user server-start instructions when you can run it.

Before making substantial visual changes, use the Product Design plugin's `get-context` skill when the visual source is unclear or no longer matches the current goal. When the user gives durable prototype-specific design feedback, preferences, or decisions, record them in `AGENTS.md`.

When implementing from a selected generated mock, treat that image as the source of truth for layout, component anatomy, density, spacing, color, typography, visible content, and hierarchy.

## Design Context

- Selected visual direction: Control Tower.
- Preserve the full-viewport cinematic hero, oversized Barlow Condensed headline, deep teal / yellow identity, visible supply-chain routes and quiet Notion-style content sections.
- Korea and Japan market expertise should stay concrete and prominent without turning the brand into a generic logistics company.
- Localization is complete-content, not navigation-only: Japanese, Chinese and Korean modes must not fall back to English body copy. Brand names, certification acronyms and messaging-platform names may remain unchanged.
- Core Services has a dedicated `/services` route; keep header, mobile navigation and footer links pointed there.
